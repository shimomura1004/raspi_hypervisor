# ハイパーバイザの起動
- `./run.sh`
- VMM(virtual macine manager) OSS がコンソール4で動くので切り替える

# ハイパーバイザの終了
- `Ctrl-a` -> `x` (QEMU の終了)

# ハイパーバイザの操作方法
|キー操作     |動作                        |
|------------|----------------------------|
|`?` -> `0-9`|コンソール切り替え            |
|`?` -> `t`  |タイマ情報の表示              |
|`?` -> `l`  |各 vCPU 上で動く VM 情報を表示|

# VMM のコマンド
|コマンド                          |動作                        |
|---------------------------------|----------------------------|
|`new` + `OS イメージのファイルパス`|指定した OS をゲストとして起動|
|`kill`                           |未サポート                   |
|`list`                           |未サポート                   |
|`shutdown`                       |未サポート                   |

- ハイパーバイザが動作する qemu 環境ではファイルシステムは fs.img の中身になっている
- Makefile 内に fs.img の内容が書かれている
- たとえば ECHO OS を起動したければ、`new ECHO.ELF` と入力する
